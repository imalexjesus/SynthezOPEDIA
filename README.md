# Энциклопедия Винтажных Синтезаторов

Веб‑приложение‑энциклопедия винтажных синтезаторов и мини‑клавишных (1970‑е–1980‑е). Каталог моделей Yamaha, Casio, Bontempi с фотографиями, характеристиками и детальными описаниями.

## Технический стек

- **Фреймворк**: SvelteKit (SSR + SSG)
- **Язык**: TypeScript
- **Сборка**: Vite
- **Дизайн**: Tailwind CSS + daisyUI (тема synthwave), адаптивный интерфейс

## Запуск проекта

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Запустите сервер разработки:
   ```bash
   npm run dev
   ```

3. Откройте в браузере: `http://localhost:5173`

## Добавление новых моделей

1. Отредактируйте `src/lib/data/synths.ts`
2. Добавьте новый объект в массив `synths` (следуйте структуре `SynthModel`)
3. Для изображений укажите реальные URL (можно использовать Wikimedia Commons, Reverb, eBay)
4. Перезапустите сервер (данные загружаются при билде)

## План наполнения каталога

### Источники данных

- **Yamaha**: Wikipedia, Vintage Synth Explorer, Piano & Synth Magazine, Google Sheets с деталями PSS
- **Casio**: Wikipedia, Vintage Synth Explorer, Synthmuseum
- **Bontempi**: Wikimedia Commons, Reverb, eBay, форумы

### Сбор характеристик

- Бренд, серия, модель, год выпуска
- Форм‑фактор (micro/mini/compact/full)
- Количество клавиш, тип синтеза, полифония
- MIDI, секвенсор, автоаккомпанемент
- Габариты, питание

### Поиск изображений

- 3‑5 фото разных ракурсов (общий вид, панель, клавиатура, логотип, задняя панель)
- Предпочитать фото без водяных знаков
- Сохранять URL в массив `images`

## Визуальный стиль

- Тёмная тема с неоновыми акцентами («японский хай‑тек 70–80‑х»)
- Градиенты, мягкие подсветки, плавные переходы
- Карточки с hover‑эффектами

## Производительность

- SSR/SSG для быстрого первого рендера
- Lazy‑loading изображений
- Кеширование данных при билде
- Code splitting для тяжёлых страниц

## API интеграция

Эндпоинты:

- `GET /api/synths` — получить список моделей (`source: local|remote`)
- `PATCH /api/synths/[id]` — частично обновить поля модели

Поддерживаемые источники (в порядке приоритета):

- NocoDB (`NOCODB_*`)
- PocketBase (`POCKETBASE_*`)
- Внешний custom API (`SYNTH_API_*`)
- Локальный fallback (in-memory)

Переменные окружения (опционально):

- `SYNTH_API_BASE_URL`
- `SYNTH_API_TOKEN`
- `NOCODB_BASE_URL`
- `NOCODB_TABLE_ID`
- `NOCODB_AUTH_TOKEN` (или `NOCODB_EMAIL` + `NOCODB_PASSWORD`)
- `POCKETBASE_URL`
- `POCKETBASE_TOKEN`
- `POCKETBASE_COLLECTION`

Если переменные не заданы, API работает в локальном runtime-режиме (in-memory).
